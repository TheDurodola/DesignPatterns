# Design Patterns Practice Repository

A personal collection of implementations of classic **GoF (Gang of Four) Design Patterns**, written primarily in **Java**.  
This repository serves as a reference, learning guide, and playground for understanding how each pattern works, when to use it, and how it improves software architecture.

---

##  Purpose

- Strengthen understanding of **OOP design principles**
- Write **clean, modular, testable** code
- Build intuition for **when and why** to apply patterns
- Maintain a reusable **knowledge base** for projects & interviews

---

## Design Pattern Categories

### **Creational Patterns**
Patterns that handle object creation mechanisms.

- Factory Method  
- Abstract Factory  
- Builder  
- Prototype  
- Singleton  

### **Structural Patterns**
Patterns that define relationships between objects.

- Adapter  
- Facade  
- Decorator  
- Composite  
- Proxy  
- Flyweight  
- Bridge  

### **Behavioral Patterns**
Patterns for communication and responsibility between objects.

- Observer  
- Strategy  
- Command  
- Chain of Responsibility  
- Template Method  
- State  
- Iterator  
- Mediator  
- Visitor  
- Memento  
- Interpreter
